Инструкции по работе с системой

В системе есть два типа пользователей.

Менеджер работает с журналом заказов: создаёт заказы, меняет статусы, назначает клиентов, редактирует технологическую схему. Раздел «Справочники» ему недоступен — при попытке открыть его система вернёт ошибку доступа.

Администратор может всё то же самое, плюс имеет полный доступ к справочникам: технологические операции, материалы, номенклатура продукции, параметры операций и список сотрудников. Создавать и удалять сотрудников может только администратор.

Чтобы открыть раздел, нажмите «Справочники» в верхней панели навигации, затем выберите «Сотрудники».

Нажмите «Создать сотрудника» над таблицей. Заполните поля:

  1. ID сотрудника — числовой идентификатор, который используется как логин при входе. Должен быть уникальным. Рекомендуется использовать табельные номера.
  2. Фамилия, имя, отчество — фамилия и имя обязательны, отчество можно не заполнять.
  3. Роль — менеджер или администратор. От выбора зависит доступ к разделам системы.
  4. Пароль — хранится в зашифрованном виде. После сохранения восстановить его нельзя, только задать новый через редактирование. Передайте пароль сотруднику напрямую.

После заполнения нажмите «Создать сотрудника» — новая запись появится в таблице.

Нажмите «Изменить» напротив нужной строки в таблице. Все поля доступны для редактирования, включая роль и пароль.

Поле пароля при открытии формы пустое. Даже если меняете только другие данные — пароль всё равно нужно ввести заново. Система требует его при любом обновлении записи.

Кнопка «Удалить» находится на странице редактирования. Если к сотруднику привязаны заказы, удаление заблокируется. Сначала переназначьте менеджера во всех связанных заказах, затем повторите.

Кнопка «Войти» расположена в правом углу верхней панели на всех страницах системы.

  1. Нажмите «Войти» — откроется страница авторизации.
  2. В поле «Логин» введите числовой ID (не имя и не почта — именно цифровой идентификатор, который задал администратор при создании аккаунта).
  3. В поле «Пароль» введите пароль, выданный администратором.
  4. Нажмите «Войти» — система перенаправит вас в журнал заказов.

Нажмите красную кнопку «Выйти» в правом углу верхней панели. Система завершит сессию и покажет страницу подтверждения выхода. Оттуда можно сразу вернуться к форме входа, нажав «Войти снова».

  1. В поле логина введено имя или фамилия — поле принимает только цифры, такой ввод сразу вызовет ошибку с пояснением.
  2. ID верный, но сотрудник не найден — администратор ещё не создал аккаунт, либо введён неверный номер.
  3. ID верный, но пароль неверный — восстановить пароль самостоятельно нельзя. Обратитесь к администратору: он откроет карточку сотрудника и задаст новый пароль через форму редактирования.

Технологическая операция — это отдельный этап производственного процесса: резка, печать, фотовывод, ламинация и т.д. Операции используются для построения технологических схем заказов — каждый узел дерева в схеме является одной операцией.

Тип обработки операции влияет на расчёт количества: система по-разному считает количество листов на каждом этапе в зависимости от выбранного типа. Поэтому важно указывать его точно.

Раздел доступен только администраторам. Перейдите в «Справочники» → «Технологические операции».

Нажмите «Создать операцию» над таблицей. Заполните поля:

  1. Название — например, «Офсетная печать» или «Резка на гильотине». Отображается в технологических схемах заказов, избегайте сокращений.
  2. Тип работы — Препресс, ДоПечать, Печать или ПослеПечать. Определяет производственный этап и используется при группировке в PDF-документе.
  3. Обработка — логика расчёта количества:
    • Стандарт — количество считается по тиражу и проценту брака.
    • Печать — учитывает цветность: количество прогонов зависит от числа красок.
    • Резка — делит тираж на части по коэффициенту резки.
    • Фотовывод — количество зависит только от числа цветов, не от тиража.
  4. Цена — стоимость операции в рублях для расчёта итоговой суммы заказа.

Нажмите «Создать операцию» — она появится в таблице и станет доступна в схемах заказов.

Нажмите «Изменить» напротив нужной строки, внесите изменения и нажмите «Изменить».

Операции в справочнике и внутри заказов хранятся раздельно — при добавлении в заказ создаётся копия. Изменение записи в справочнике не затронет существующие заказы, только новые.

Кнопка «Удалить» находится на странице редактирования. Если операция привязана к активным заказам, удаление заблокируется — сначала уберите её из всех связанных заказов.

Материал — расходный ресурс, используемый при выполнении операции: бумага, плёнка, картон, краска и т.д. Каждый материал привязывается к конкретной операции внутри заказа.

Когда менеджер открывает параметры операции в технологической схеме, он указывает, какой материал задействован и в каком количестве на единицу продукции. Система автоматически считает общий расход и включает его стоимость в итоговую сумму заказа.

Раздел доступен только администраторам. Перейдите в «Справочники» → «Материалы».

Нажмите «Создать материал» над таблицей. Заполните поля:

  1. Название — например, «Мелованная бумага 130 г/м²» или «Плёнка матовая». Чем конкретнее, тем удобнее менеджеру ориентироваться при выборе.
  2. Цена — стоимость за единицу в рублях. Принимает дробные значения, разделитель — точка. Умножается на количество при расчёте стоимости заказа.

Нажмите «Создать материал» — запись появится в таблице и станет доступна в схемах заказов.

Нажмите «Изменить» напротив нужной строки, внесите изменения и нажмите «Изменить».

Изменение цены затронет только новые расчёты. Уже рассчитанные заказы хранят зафиксированную сумму — пересчёт произойдёт только при повторном запуске расчёта стоимости в конкретном заказе.

Кнопка «Удалить» на странице редактирования. Если материал используется в операциях — удаление заблокируется. Сначала уберите его из всех операций, затем удаляйте из справочника.

Это список изделий, которые производит типография: визитки, буклеты, листовки, каталоги и т.д. Когда менеджер создаёт новый заказ, он выбирает изделие именно из этого списка — и его название становится названием заказа.

К каждой позиции можно прикрепить изображение. Оно будет отображаться в технологической схеме заказа в блоке «Изображение» для визуальной сверки.

Раздел доступен только администраторам. Перейдите в «Справочники» → «Номенклатура продукции».

Нажмите «Создать номенклатуру» над таблицей. Заполните поля:

  1. Название — отображается в выпадающем списке при создании заказа, формулировка должна быть понятна менеджерам.
  2. Изображение (необязательно) — загрузите фото или макет изделия в формате JPG, PNG или GIF. После выбора файла появится предварительный просмотр.

Нажмите «Создать номенклатуру» — позиция появится в таблице.

Нажмите «Изменить» напротив нужной строки. Можно поменять название и заменить изображение. Если загрузить новый файл — старый автоматически удалится с сервера. Если оставить поле файла пустым — изображение останется прежним.

Нажмите «Сохранить» для применения изменений.

Кнопка «Удалить» заблокируется, если к позиции привязаны заказы. Сначала удалите или перепривяжите все связанные заказы.

Параметр операции — характеристика, задаваемая для конкретной операции в заказе: цветность, процент брака, количество частей при резке и т.д.

Справочник хранит только названия параметров без значений. Значения задаются отдельно для каждой операции в каждом заказе. Например, параметр «Цветность» — одна запись в справочнике, а значения в разных заказах могут быть «4+0» или «1+1».

Параметры напрямую влияют на расчёт количества. Процент брака, цветность и количество частей при резке должны иметь точные названия из справочника — иначе система их не распознает и расчёт даст неверный результат.

Нажмите «Создать параметр» над таблицей. В поле «Название параметра» введите однозначное название — оно будет отображаться в таблице параметров внутри операции заказа.

Нажмите «Создать параметр» — запись появится в таблице и станет доступна для добавления к операциям в технологических схемах.

Нажмите «Изменить» напротив нужного параметра, измените текст и нажмите «Изменить».

Переименование параметра обновит его отображение везде, где он уже добавлен к операциям — в базе хранится ссылка на запись, а не копия названия.

Кнопка «Удалить» на странице редактирования. Если параметр используется хотя бы в одной операции любого заказа — удаление заблокируется. Сначала удалите его из всех операций вручную.

Стартовая страница журнала. Показывает заказы с любым статусом без фильтрации — удобно использовать как общий список для быстрого поиска, когда не помните текущий статус заказа.

Таблица показывает: номер заказа, дату создания, ответственного менеджера, название изделия, статус и кнопку «Перейти». По нажатию система открывает страницу заказа в соответствии с его статусом.

Над таблицей — строка поиска (по названию заказа) и сортировка по номеру, дате или алфавиту. Выбранная сортировка сохраняется в адресной строке.

Заказы со статусом «шаблон». Шаблон — заготовка, которую можно многократно копировать. Удобно, когда типография регулярно выпускает один и тот же продукт: вместо того чтобы каждый раз строить дерево операций с нуля, менеджер копирует шаблон.

В таблице две кнопки: «Перейти» открывает паспорт шаблона, «Скопировать» создаёт новый заказ со статусом «расчет» на основе этого шаблона и перенаправляет на паспорт нового заказа.

Заказы в активной работе. Статус «расчет» означает, что заказ оформляется: менеджер заполняет паспорт, строит технологическую схему, задаёт параметры операций и рассчитывает стоимость.

По нажатию «Перейти» открывается паспорт в полном режиме редактирования. Из паспорта доступна технологическая схема с деревом операций. Отсюда заказ можно перевести в статус «проверен», «шаблон» или «на удаление».

Заказы со статусом «проверен». Это финальная стадия: схема сформирована, стоимость рассчитана, данные проверены.

Паспорт открывается только для чтения — редактировать поля и добавлять операции нельзя. Из паспорта можно вернуть заказ в расчёт или перевести в шаблон.

Промежуточный статус перед окончательным удалением. Заказ остаётся в системе и виден здесь до тех пор, пока администратор не удалит его.

Паспорт в этом разделе позволяет редактировать даты, менеджера и клиента. Кнопка «Удалить» доступна только здесь и только администраторам — после нажатия заказ исчезает полностью без возможности восстановления. Если удаление ошибочное, заказ можно вернуть в другой статус через кнопки в паспорте.

Нажмите «Новый» в меню журнала заказов. На странице — форма с одним полем: выпадающий список «Название заказа» с позициями из справочника номенклатуры.

  1. Выберите нужное изделие и нажмите «Создать заказ».
  2. Система автоматически создаёт заказ с уникальным номером и статусом «расчет», формирует начальную технологическую схему и открывает её.
  3. Дальше добавляйте нужные операции, задавайте параметры, привязывайте материалы.
  4. Паспортные данные, даты и ответственный менеджер заполняются на странице паспорта — перейдите туда через кнопку «Паспорт заказа» в верхнем меню.
Если нужного изделия нет в списке — оно не заведено в справочнике номенклатуры. Обратитесь к администратору.

На странице заказа в верхнем меню три вкладки: «Паспорт заказа», «Технологическая схема» и «Материалы заказа».

Страница схемы делится на две части:

  • Слева — дерево операций. Каждый узел — одна операция с количеством на выходе и названием. Рядом с каждой — три кнопки: добавить дочернюю, удалить, открыть параметры.
  • Справа — панель управления: форма расчёта стоимости, блок материалов и изображение изделия из номенклатуры.
Для заказов со статусом «расчет» дерево интерактивное. Для проверенных заказов — только просмотр параметров.

Добавить операцию: нажмите кнопку добавления рядом с нужной операцией — справа откроется список операций из справочника с указанием типа работы. Выберите нужную и нажмите «Добавить операцию». Система создаёт копию и прикрепляет её как дочерний узел.

Изменить параметры: нажмите кнопку с прицелом рядом с операцией — справа откроется панель редактирования. Там можно менять расшифровку, параметры, материал и его количество.

Удаление работает рекурсивно — если удалить операцию с дочерними, они все удалятся вместе с ней. Подтверждение не запрашивается, будьте внимательны.

Откройте параметры нужной операции кнопкой с прицелом. В нижней части панели — блок «Материал» с двумя полями: выпадающий список материалов и поле количества.

  1. Выберите материал из списка.
  2. Укажите количество на единицу продукции.
  3. Нажмите «Сохранить изменения» — сводная таблица материалов обновится.
К одной операции можно привязать только один материал. Чтобы заменить — выберите другой из списка и сохраните.

В панели параметров операции — таблица уже добавленных параметров с их значениями и форма добавления нового: выпадающий список из справочника и кнопка «Добавить».

  1. Выберите параметр из списка и нажмите «Добавить» — он появится в таблице с пустым полем значения.
  2. Введите значение прямо в ячейку таблицы (цветность можно записать как «4+0», числа — цифрами).
  3. Нажмите «Сохранить изменения».
Параметры, влияющие на расчёт количества (процент брака, цветность, количество частей при резке), должны иметь точные названия из справочника. Иначе система их не распознает.

Форма расчёта находится в правой части страницы технологической схемы, в верхней карточке. Поле для тиража, кнопка «Рассчитать стоимость» и поле итоговой суммы, которое заполняется автоматически.

  1. Введите тираж и нажмите «Рассчитать стоимость».
  2. Система обходит все операции дерева и для каждой считает количество на входе и выходе с учётом типа обработки, брака, цветности или коэффициента резки.
  3. Суммируются стоимости всех операций и материалов — итог появляется в поле «Стоимость» и сохраняется в паспорте заказа.
  4. Сводная таблица материалов тоже обновится с учётом нового тиража.
Если какой-то элемент (цена операции, количество материала, параметр) не заполнен — система посчитает без него. Проверьте полноту данных перед расчётом.

Нажмите «Сгенерировать PDF» — откроется страница предварительного просмотра документа.

В документ попадает: дерево заказа с количествами на каждой операции и четыре блока операций, сгруппированных по типу работы (Препресс, ДоПечать, Печать, ПослеПечать). Для каждой операции — количество на входе и выходе, брак и все заданные параметры с значениями.

Чтобы распечатать или сохранить как PDF, нажмите «Печать страницы» в нижней части страницы — откроется стандартное диалоговое окно браузера. Кнопки навигации при печати скрываются автоматически.

Паспорт доступен из журнала заказов (кнопка «Перейти») и из технологической схемы (кнопка «Паспорт заказа» в верхнем меню).

Страница состоит из двух карточек:

  • Левая — основные данные заказа: номер, название изделия, статус, тираж и рассчитанная сумма.
  • Правая — формы для заполнения деталей.

Внизу страницы — кнопки смены статуса заказа.

Каждая форма сохраняется независимо — изменение одного поля не затрагивает остальные.

  1. Дата и время создания — два поля рядом, сохраняются парой одним нажатием кнопки «Сохранить».
  2. Дата и время готовности — работает так же. Это планируемая дата выдачи для контроля сроков.
  3. Менеджер — выпадающий список всех сотрудников. Выберите ответственного и нажмите «Сохранить».
  4. Клиент — выпадающий список клиентов. Каждая строка содержит ID, название, контакты и адрес. Выберите и нажмите «Сохранить».

Внизу страницы паспорта расположены кнопки: «В шаблон», «Расчет», «В проверку» и «На удаление».

Нажатие на любую из них немедленно меняет статус и перенаправляет на соответствующую версию паспорта.

Для проверенных заказов паспорт открывается только для чтения — редактирование полей недоступно. Чтобы вернуться к редактированию, переведите заказ обратно в статус «расчет».
